# Release Management

The page describes the release management process of SuperSport Web Components.

## Release Strategy

SuperSport Web Components follow the well-known NPM Semantic Versioning strategy and produce the following type of releases:

| Type    | Description                                                                                                                                  |
| :------ | :------------------------------------------------------------------------------------------------------------------------------------------- |
| `Patch` | Includes backward compatible bug fixes and increments the third digit, e.g. 1.0.1.                                                           |
| `Minor` | Includes backward compatible new features and increments the middle digit and resets the last digit to zero, e.g. 1.1.0.                     |
| `Major` | Includes changes that break backward compatibility and increments the first digit and resets the middle and last digits to zero, e.g. 2.0.0. |
